# Conducting User Interviews

Use this skill when preparing user interviews, writing interview guides, coaching interview technique, or converting interview notes into a clean debrief.

Source note: inspired by RefoundAI Lenny Skills `conducting-user-interviews` (MIT). This version is written for this kit.

## Workflow

1. Define the research decision, learning goals, target segment, and disqualifying criteria.
2. Recruit for recent relevant behavior, not generic demographics.
3. Write questions that ask about past behavior, actual workflows, triggers, constraints, and alternatives.
4. Run the session:
   - Set context and permission.
   - Start broad, then ask for the last real instance.
   - Probe for sequence, tools, people involved, cost, and workaround.
   - Avoid pitching the solution until the end, if at all.
5. Debrief immediately: strongest evidence, surprising moments, contradictions, and follow-up candidates.
6. Prepare synthesis input with quotes tied to participant segment and context.

## Output

- Interview objective
- Participant criteria
- Screener questions
- Interview guide
- Moderator notes
- Debrief template
- Evidence capture table
- Follow-up plan

## Guardrails

- Avoid "would you use" and "do you like" as primary evidence.
- Do not lead with mockups before understanding the current behavior.
- Separate participant quotes from interviewer interpretation.
