# PM Launch Retro

Use this skill for launch checklists, go/no-go reviews, rollout planning, metric observation, release communications, and post-launch retrospectives.

## Workflow

1. Clarify launch scope, audience, rollout stages, success metrics, guardrail metrics, and rollback owner.
2. Build pre-launch, launch-day, and post-launch checklists.
3. Include analytics, support, documentation, training, privacy, reliability, and operational readiness.
4. Define observation windows and decision thresholds.
5. For retro, compare expected vs actual results and capture follow-up owners.

## Output

- 上线范围与目标
- Go/no-go checklist
- 灰度与回滚计划
- 数据观察计划
- 客服/销售/运营准备
- 上线沟通稿要点
- 复盘模板与行动项

## Guardrails

- Do not declare launch-ready if metrics, rollback, or support paths are undefined.
- Call out dependencies that need explicit owner confirmation.
- Separate launch blockers from nice-to-have cleanup.
