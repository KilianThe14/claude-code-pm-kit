# PM Review

Use this skill when the user asks to review, critique, challenge, red-team, or improve a PRD, roadmap, proposal, experiment, or launch plan.

## Workflow

1. Read for intended outcome, user value, metrics, scope, dependencies, and launch readiness.
2. Lead with the highest-risk findings, not a general summary.
3. Check whether the proposal connects problem, evidence, solution, metric, and execution.
4. Identify missing non-goals, edge cases, operational costs, data needs, and rollback plans.
5. Suggest concrete changes or questions that unblock the next review.

## Output

- 严重问题
- 中等问题
- 小问题或表达改进
- 待确认问题
- 建议修改后的关键段落或清单

## Guardrails

- Be direct but grounded in the provided text.
- Do not request changes for personal style preferences unless clarity or decision quality suffers.
- If evidence is missing, state what evidence would be sufficient.
