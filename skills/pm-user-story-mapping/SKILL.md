# PM User Story Mapping

Use this skill for user story mapping, journey decomposition, MVP slicing, release planning, task flows, and acceptance criteria.

## Workflow

1. Clarify persona, job-to-be-done, start/end state, channels, and constraints.
2. Map the backbone: discover, evaluate, act, recover, repeat, or a domain-specific flow.
3. Break each step into user stories and acceptance criteria.
4. Slice MVP by highest-value path and explicit non-goals.
5. Identify dependencies, edge cases, and later release candidates.

## Output

- 用户目标与核心任务
- 用户旅程骨架
- 用户故事列表
- MVP 边界
- V1/V2/VNext 拆分
- 验收标准
- 风险与待确认项

## Guardrails

- Keep stories tied to user outcomes, not internal implementation tasks.
- Do not include every edge case in MVP unless it blocks the core journey.
- Mark dependencies that affect release sequencing.
