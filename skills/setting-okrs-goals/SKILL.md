# Setting OKRs Goals

Use this skill when the user needs OKRs, product goals, metric trees, quarterly planning goals, or a connection between strategy, initiatives, and measurable outcomes.

Source note: inspired by RefoundAI Lenny Skills `setting-okrs-goals` (MIT). This version is written for this kit.

## Workflow

1. Clarify strategy, audience, planning period, baseline metrics, constraints, and accountable team.
2. Define objectives as directional outcomes, not feature delivery.
3. Draft key results that are measurable, time-bound, and paired with baseline or target logic.
4. Build a metric tree: input metrics, output metrics, guardrails, and diagnostic metrics.
5. Link initiatives to KRs and identify gaps where goals lack credible work.
6. Set review cadence, owner, reporting format, and decision rules for changing course.

## Output

- Strategy context
- Objective options
- Key results with baselines and targets
- Metric tree
- Initiative-to-OKR mapping
- Risks and guardrails
- Review cadence
- Open data questions

## Guardrails

- Avoid feature-shipping KRs unless delivery itself is the measurable bottleneck.
- Do not invent baselines; mark them as needed data.
- Include guardrails so growth goals do not degrade quality, trust, or unit economics.
