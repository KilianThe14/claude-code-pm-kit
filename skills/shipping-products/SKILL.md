# Shipping Products

Use this skill when the user needs help getting a product shipped: execution plan, scope cut, blocker handling, launch readiness, rollout decision, team operating cadence, or post-launch learning.

Source note: inspired by RefoundAI Lenny Skills `shipping-products` (MIT). This version is written for this kit.

## Workflow

1. Clarify ship goal, target date, must-have outcome, current status, owners, and blockers.
2. Split scope into:
   - Core user value.
   - Risk reducers.
   - Launch requirements.
   - Nice-to-have polish.
   - Later follow-ups.
3. Build an execution plan with milestones, dependencies, decision points, and escalation rules.
4. Run trade-off checks: what can be cut, sequenced, faked, manualized, or delayed without breaking user value.
5. Prepare launch readiness: QA/UAT, analytics, support, comms, rollback, privacy/compliance, and monitoring.
6. After launch, compare observed outcomes to goals and capture follow-up decisions.

## Output

- Shipping objective
- Current status and blockers
- Scope trade-off table
- Milestone plan
- Decision and escalation plan
- Launch readiness checklist
- Rollout and rollback plan
- Post-launch learning plan

## Guardrails

- Shipping faster should not mean hiding unresolved quality, compliance, or rollback risks.
- Do not cut the smallest item by effort if it protects core user value.
- Make decision owners explicit for every blocker.
